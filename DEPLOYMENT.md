# 部署和手机离线使用

当前开发地址 `http://192.168.x.x:5174/` 只适合局域网测试。手机离开同一个 WiFi、电脑关机、开发服务器停止后，这个地址都会失效。

要做到手机随时打开，需要把项目部署到 HTTPS 地址，然后从手机浏览器访问该地址并添加到主屏幕。PWA 的 Service Worker 需要 HTTPS 才能在手机上可靠工作。

## 推荐部署方式

### Vercel

1. 把项目推到 GitHub 仓库。
2. 在 Vercel 导入仓库。
3. 构建命令使用：

```bash
npm run build
```

4. 输出目录使用：

```bash
dist
```

5. 部署完成后，用手机打开 Vercel 提供的 HTTPS 地址。

### Netlify

1. 把项目推到 GitHub 仓库。
2. 在 Netlify 导入仓库。
3. 构建命令使用：

```bash
npm run build
```

4. 发布目录使用：

```bash
dist
```

5. 部署完成后，用手机打开 Netlify 提供的 HTTPS 地址。

## 手机安装

iPhone：

1. 用 Safari 打开线上 HTTPS 地址。
2. 点击分享按钮。
3. 选择“添加到主屏幕”。
4. 从桌面图标打开应用。

Android：

1. 用 Chrome 打开线上 HTTPS 地址。
2. 点击菜单。
3. 选择“安装应用”或“添加到主屏幕”。
4. 从桌面图标打开应用。

## 离线测试

1. 第一次必须联网打开线上 HTTPS 地址。
2. 至少切换一次“今日、打卡、训练、建议、周报”，让资源完成缓存。
3. 关闭网络。
4. 从手机桌面图标再次打开应用。

本应用数据保存在手机浏览器本地存储中，不需要后端。换手机、清浏览器数据或卸载浏览器后，本地数据可能丢失，建议定期在周报页导出 JSON。
